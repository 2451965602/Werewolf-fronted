## Why

当前狼人杀前端把“是否存在进行中对局”直接映射为首页跳转结果，导致用户在首次进入时无法稳定看到开局入口；同时现有页面在 1920×1080 下首屏信息密度失衡、AI 发言与投票流程展示不完整，整体观感仍偏测试演示页。现在需要把首页入口、观战信息层级和正式产品化视觉一起拉齐，避免继续在当前结构上零碎修补。

## What Changes

- 将首页从“纯 pregame 说明页”改为真正的 Lobby：首页始终先展示开局入口。
- 当服务端存在进行中对局时，在 Lobby 首屏展示高优先级“继续当前对局”卡片，而不是自动跳转到控制台。
- 重构对局主界面信息层级，改为“阶段焦点 + 时间线主舞台 + 玩家态势 + 操作区”。
- 补齐 AI 发言、系统播报、投票动作/结果的时间线可见性。
- 使用 shadcn 组件重构主要 UI 容器与信息卡片，降低测试页观感并收紧 1080p 首屏布局。

## Capabilities

### New Capabilities
- `game-lobby-experience`: 定义 Lobby 首页、继续进行中对局卡片与显式进入对局的用户流程。

### Modified Capabilities
- `werewolf-web-frontend`: 调整单入口页面的进入规则、对局信息布局和时间线展示要求。

## Impact

- 受影响代码主要在 `App.tsx`、`use-game-console.ts`、`view-model.ts` 和 `features/game/components/*`。
- 不新增后端 API；继续使用现有游戏状态、消息、健康检查与推进接口。
- 需要新增/引入更多 shadcn 组件作为源代码组件。
